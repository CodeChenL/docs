## 使用 `rsdk`

[`rsdk`](https://github.com/RadxaOS-SDK/rsdk) 是目前 RadxaOS 的生成环境，其有以下几个特点：

1. 生成环境容器化，无需额外配置生成依赖
2. 模块化的生成代码，修改方便
3. 支持 x86、ARM64 平台上运行

### 这里简单介绍如何安装并使用 `rsdk`

<Tabs queryString="method">
<TabItem value="devcontainers">
1. 首先，请安装所需的依赖项：

```
sudo apt-get update
sudo apt-get install git qemu-user-static binfmt-support
sudo apt-get install npm docker.io
sudo usermod -a -G docker $USER
```

2. 使用以下命令克隆项目 git 并安装  devcontainer

```
git clone --recurse-submodules https://github.com/RadxaOS-SDK/rsdk.git
cd rsdk
npm install @devcontainers/cli
export PATH="$PWD/src/bin:$PWD/node_modules/.bin:$PATH"
rsdk devcon up
rsdk devcon
```

:::warning
推荐优先使用开发容器进行系统镜像构建。  
使用原生编译方法，由于构建依赖版本无法控制可能出现意料外的错误，如 `jsonnet` 不支持 `isEmpty` 方法。  
此方法仅在 Ubuntu 24.04 系统上进行了测试。
:::

1. 首先，请安装所需的依赖项：

```
sudo apt-get update
sudo apt-get install bdebstrap binfmt-support dosfstools gdisk git jsonnet jq libguestfs-tools qemu-user-static
```

2. 使用以下命令克隆项目 git

```
git clone --recurse-submodules https://github.com/RadxaOS-SDK/rsdk.git
cd rsdk
export PATH="$PWD/src/bin:$PWD/node_modules/.bin:$PATH"
```

```

```

仓库详细使用，请参考 [rsdk](https://radxaos-sdk.github.io/rsdk/) 页面。

如需要在 `rsdk` 基础上进行二次开发，则请继续阅读文档 [Build customization](https://radxaos-sdk.github.io/rsdk/tutorials/customize.html)。

### 常见问题

1. 开发容器设置暂停并提示：You might be rate limited by GitHub。您可能受到 GitHub 的速率限制。请按照输出中列出的说明进行操作。
2. 启动开发容器失败。请编辑 .devcontainer/devcontainer.json，并调整 runArgs 属性，或者尝试使用 [native 方法构建](rsdk?method=native)。
